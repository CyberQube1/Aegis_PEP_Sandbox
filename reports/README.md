# Frozen Report Artifacts

This directory contains both the curated table inputs used by the paper and the frozen proof artifacts from the Aegis paper evaluation sandbox.

## Primary frozen proof artifacts

- `frozen_10_run_folder/`: copied from the internal `10 Run Folder` report pack. This is the full frozen 10-run evidence package, including per-run outputs, comparison reports, traces, Senate status artifacts, provenance artifacts, leakage artifacts, and table/report inputs.
- `frozen_one_run_folder/`: copied from the internal `One Run Folder` report pack. This is the full frozen single-run evidence package, including per-run outputs, comparison reports, traces, Senate status artifacts, provenance artifacts, leakage artifacts, and table/report inputs.

These folders are evidence artifacts. They are not generated by the public offline mode, and they do not include the Aegis kernel or production trust infrastructure.

The frozen folders may contain synthetic evidence identifiers such as paper policy-bundle IDs, baseline-release IDs, decision IDs, trace IDs, and Senate tally IDs. These identifiers are part of the report chain. They are not credentials, keys, certificates, tokens, endpoints, production mesh IDs, or production trust material.

## Curated paper/table summaries

- `10_run/`: normalized 10-run report summaries and table inputs used by the manuscript and supplementary appendix.
- `single_run/`: normalized single-run report summaries and table inputs used by the manuscript and supplementary appendix.

The curated summaries are easier to inspect, while the frozen proof-artifact folders preserve the fuller report structure used to support the claim chain.
